## 要約
- 発生条件（1行）
  - インストーラー、更新、またはファイルアクセス操作で指定されたファイルが存在しない、またはアクセス不能なときに「エラー 0x80070002: 要求されたファイルが見つかりません」が返る。
- 影響（1行）
  - インストール/更新失敗、アプリ起動不能、スクリプト実行停止などの直接的な処理中断を引き起こす。
- 緊急度（中、理由1行）
  - 中：ユーザー作業を止めるが、原因が権限・パス・ロック・セキュリティ設定のいずれかで対処可能なため。

## よくある原因（優先度順）
1. 指定パスが間違っている（フルパスのスペル/拡張子欠損）
2. 所有権/ACLでアクセスが拒否されている
3. ダウンロードや展開時にZone.Identifier/MOTWでブロックされている
4. ファイルが別プロセスにロックされている（アンチウイルス/同期クライアント/別ユーザー）
5. AppLocker / ソフトウェア制限ポリシー(SRP) / SmartScreen などの実行制御によるブロック

## 60秒で確認（最短手順）
- 管理者で起動 → 再試行
- 現在のACLを確認：
  ```cmd
  icacls "C:\Path\To\Folder"
  ```
- 自分の特権を確認：
  ```cmd
  whoami /groups & whoami /priv
  ```
- Defenderのブロック/履歴とCFA有効状態を確認：
  ```powershell
  Get-MpThreatDetection -ErrorAction SilentlyContinue | Select Timestamp, Action, Resources | Select -First 5
  Get-MpPreference | Select EnableControlledFolderAccess
  ```
- TEMPフォルダーの権限を確認：
  ```cmd
  echo %TEMP%
  icacls "%TEMP%"
  ```
- GPOの影響をスナップショット：
  ```cmd
  gpresult /h "%USERPROFILE%\Desktop\gp.html"
  ```

## 5分で解決（実務フロー）
1) 対象を特定 → 所有権/権限を付与（GUIかコマンド）
2) 制御されたフォルダーアクセスやEDRの許可登録、MOTW(Zone.Identifier)の解除を実施
   - PowerShell（ブロック解除）:
     ```powershell
     Unblock-File -Path "C:\Path\to\installer-or-script.exe"
     ```
   - GUI: 「プロパティ → ブロックの解除」にチェックして再試行
3) ロック中ならリブート/セーフモード/保持プロセスの安全な終了（resmonでハンドル特定）

## 根本対策（恒久策）
- 所有権と継承の是正（必要最小限の範囲で実施）
- Defender/EDRに対する正式な許可リストと例外ポリシーを運用化
- GPO/EDR配下でのソフト配布フローと申請手順を整備
- 共有ドライブやZIP直実行を避け、ローカル（例: C:\Temp）にコピーしてから実行する運用

## 詳細手順（Windows 10/11）
1. GUIでの設定変更（対象フォルダー右クリック → プロパティ → セキュリティ → 詳細 → 所有者変更 → 継承設定）
2. コマンド例（管理者で実行）
   ```cmd
   takeown /F "C:\Path\To\Folder" /R /D Y
   icacls "C:\Path\To\Folder" /grant Administrators:F /T
   ```
   ロールバック例（SYSTEMに戻す）:
   ```powershell
   $p = "C:\Path\To\Folder"
   icacls $p /setowner "NT AUTHORITY\SYSTEM" /T
   ```
   付け過ぎた権限を既定に戻す（継承/ACLの修復）:
   ```cmd
   icacls "C:\Path\To\Folder" /inheritance:e
   icacls "C:\Path\To\Folder" /reset
   ```
3. Defender: 状態確認と許可追加
   - 状態確認:
     ```powershell
     Get-MpPreference | Select EnableControlledFolderAccess, ControlledFolderAccessAllowedApplications
     ```
   - 許可追加:
     ```powershell
     Add-MpPreference -ControlledFolderAccessAllowedApplications "C:\Path\app.exe"
     ```
4. ファイルがロック中なら（見つけ方）
   - 「Win + R」→「resmon」→CPUタブ→"関連付けられたハンドル"でパスを検索→該当プロセスを終了、または再起動／セーフモードで作業。

⚠ 注意: 一時的にアンチウイルスやCFAを無効化して作業する場合は作業中のみ行い、完了後に必ず再有効化して設定を元に戻す（ロールバック: Defenderを有効化/許可設定を削除しない場合は復元）。

## よくある落とし穴（やってはいけない）
- Windows/Program Files/System32 直下への広範な所有権変更
- 共有フォルダーに Everyone:フルを付与すること
- 根拠なくアンチウイルスを恒久的に無効化すること

## 再発防止チェックリスト
- 標準ユーザー運用を徹底、必要時のみ昇格する仕組みを用意
- インストーラーはダウンロード直後に実行してZone情報を確実に確認
- 変更手順を文書化し、レビューを行う
