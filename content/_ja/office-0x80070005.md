## 要約
- 発生条件（1行）  
  Office 更新実行中にアクセス拒否でインストール/アップデートが完了せず、エラーコード 0x80070005 が返る。  
- 影響（1行）  
  Office の更新が停止し、セキュリティ修正や機能更新が適用されない。  
- 緊急度（高/中/低、理由1行）  
  中。ユーザー作業を阻害しセキュリティ欠陥を残すため、迅速な権限確認が必要。

## よくある原因（優先度順）
1. 権限不足（インストーラーが書き込み/実行するフォルダーに対するACL不足）  
2. 所有権が別ユーザー（SYSTEM/別ユーザーが所有していてアクセス不可）  
3. セキュリティ製品（Defender/EDR/Controlled Folder Access）がファイルやプロセスをブロック  
4. ファイルロック（別プロセスがインストーラー/一時ファイルを開いている）  
5. AppLocker / ソフトウェア制限ポリシー(SRP) / SmartScreen 等の実行制御によるブロック

## 60秒で確認（最短手順）
- 管理者で起動 → 再試行
- 現在のACLを確認：
  ```cmd
  icacls "C:\Path\To\Folder"
  ```
- 自分の特権を確認：
  ```cmd
  whoami /groups & whoami /priv
  ```
- Defenderのブロック/履歴とCFA有効状態を確認：
  ```powershell
  Get-MpThreatDetection -ErrorAction SilentlyContinue | Select Timestamp, Action, Resources | Select -First 5
  Get-MpPreference | Select EnableControlledFolderAccess
  ```
- TEMPフォルダーの権限を確認：
  ```cmd
  echo %TEMP%
  icacls "%TEMP%"
  ```
- GPOの影響をスナップショット：
  ```cmd
  gpresult /h "%USERPROFILE%\Desktop\gp.html"
  ```

## 5分で解決（実務フロー）
1) 対象を特定 → 所有権/権限を付与（GUIまたはコマンドで実行）  
2) 制御されたフォルダーアクセスやEDRの許可登録、MOTW(Zone.Identifier)の解除を実施  
   - PowerShell（ブロック解除）:
     ```powershell
     Unblock-File -Path "C:\Path\to\installer-or-script.exe"
     ```
   - GUI: 「プロパティ → ブロックの解除」にチェック  
3) ロック中ならリブート/セーフモード/保持プロセスの安全な終了

## 根本対策（恒久策）
- 所有権と継承を局所的に是正（システムフォルダーへ広範に変更しない）  
- Defender/EDR: 許可アプリ/例外ポリシーを整備して正当なインストーラーを除外  
- GPO/EDR配下の申請フローを運用化し、例外付与手順を文書化  
- 共有や圧縮ファイル直からの実行を避け、ローカル（例: C:\Temp）へコピーしてから実行

## 詳細手順（Windows 10/11）
1. GUIでの設定変更（プロパティ → セキュリティ → 詳細 → 所有者変更 → 継承有効化 → 必要な権限を付与）  
2. コマンド例（管理者で実行）
   ```cmd
   takeown /F "C:\Path\To\Folder" /R /D Y
   icacls "C:\Path\To\Folder" /grant Administrators:F /T
   ```
   **ロールバック例**（SYSTEMに戻す）:
   ```powershell
   $p = "C:\Path\To\Folder"
   icacls $p /setowner "NT AUTHORITY\SYSTEM" /T
   ```
   **付け過ぎた権限を既定に戻す（継承/ACLの修復）**:
   ```cmd
   icacls "C:\Path\To\Folder" /inheritance:e
   icacls "C:\Path\To\Folder" /reset
   ```
3. Defender: 設定と許可アプリ  
   - 状態確認:
     ```powershell
     Get-MpPreference | Select EnableControlledFolderAccess, ControlledFolderAccessAllowedApplications
     ```
   - 許可追加:
     ```powershell
     Add-MpPreference -ControlledFolderAccessAllowedApplications "C:\Path\app.exe"
     ```
4. ファイルがロック中なら（見つけ方）  
   - Win + R → resmon → CPU → 関連付けられたハンドルでパスを検索 → 該当プロセスを終了、または再起動／セーフモードで作業。  
⚠ 注意: セキュリティ製品の無効化や広範な権限付与は作業中のみ実施し、作業完了後は必ず元に戻す。ロールバック: 無効化した設定を同じ手順で再有効化。

## よくある落とし穴（やってはいけない）
- Windows/Program Files/System32 直下への広範な所有権変更  
- 共有フォルダーに Everyone:Full を付与してしまうこと

## 再発防止チェックリスト
- 標準ユーザー運用＋必要時のみ昇格を徹底する  
- インストーラーはダウンロード直後にローカルにコピーして実行する  
- インストール手順と例外申請を文書化しレビューを行う

## 関連ケース
- Windows Update で 0x80070005 が出るときの手順  
- Office/Outlook の有効化で 0x80070005 が出るとき  
- OneDrive/同期クライアントで 0x80070005 が出るとき
