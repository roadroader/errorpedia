## 要約
- 発生条件（1行）  
  .NET 更新プログラムのインストール中にインストーラーがアクセス権/ファイルロック/実行ブロックで失敗してエラー 0x80070643 を返す。
- 影響（1行）  
  .NET ランタイム/累積更新が適用されず、アプリケーションの動作やセキュリティ修正が反映されない。
- 緊急度（中、理由1行）  
  システムは通常動作するが、セキュリティ修正や依存アプリの互換性に影響するため迅速対応が望ましい。

## よくある原因（優先度順）
1. インストーラーが書き込むフォルダーに対する所有権/ACL不足（インストーラーはSYSTEMまたは管理者権限要求）。
2. Defender/EDR のブロックや制御されたフォルダーアクセスによるブロック。
3. インストーラーファイルの「ブロック」フラグ（Zone.Identifier/MOTW）や署名問題。
4. ファイル/ハンドルが別プロセスによりロックされている（ファイルロック）。
5. AppLocker / ソフトウェア制限ポリシー(SRP) / SmartScreen などの実行制御によるブロック。

## 60秒で確認（最短手順）
- 管理者で起動 → 再試行
- 現在のACLを確認：
  ```cmd
  icacls "C:\Path\To\Folder"
  ```
- 自分の特権を確認：
  ```cmd
  whoami /groups & whoami /priv
  ```
- Defenderのブロック/履歴とCFA有効状態を確認：
  ```powershell
  Get-MpThreatDetection -ErrorAction SilentlyContinue | Select Timestamp, Action, Resources | Select -First 5
  Get-MpPreference | Select EnableControlledFolderAccess
  ```
- TEMPフォルダーの権限を確認：
  ```cmd
  echo %TEMP%
  icacls "%TEMP%"
  ```
- GPOの影響をスナップショット：
  ```cmd
  gpresult /h "%USERPROFILE%\Desktop\gp.html"
  ```

## 5分で解決（実務フロー）
1) 対象を特定 → 所有権/権限を付与（GUIまたはコマンドで特定フォルダーのみ）
2) 制御されたフォルダーアクセスやEDRの許可登録、MOTW(Zone.Identifier)の解除を実施  
   - PowerShell（ブロック解除）:
     ```powershell
     Unblock-File -Path "C:\Path\to\installer-or-script.exe"
     ```
   - GUI: インストーラーの「プロパティ → ブロックの解除」にチェック
3) ロック中ならリブート/セーフモード/保持プロセスの安全な終了（resmon でハンドル特定）
4) 再試行してインストール成功を確認

## 根本対策（恒久策）
- 所有権と継承の是正を局所で実施し、広範な権限変更を避ける。
- Defender/EDR に対して許可アプリと例外ポリシーを整備する（展開プロセスを運用化）。
- GPO/EDR 管理下の申請フローを整備し、インストーラー配布時の事前承認を行う。
- 共有や圧縮ファイル直実行を避け、必ずローカル（例: C:\Temp）にコピーしてから実行する。

## 詳細手順（Windows 10/11）
1. GUIでの設定変更：該当フォルダーを右クリック → プロパティ → セキュリティ → 詳細 → 所有者を管理者またはSYSTEMに設定 → 継承を有効にして必要最小限のアクセス権を付与。
2. コマンド例（管理者で実行）
   ```cmd
   takeown /F "C:\Path\To\Folder" /R /D Y
   icacls "C:\Path\To\Folder" /grant Administrators:F /T
   ```
   ロールバック例（SYSTEMに戻す）:
   ```powershell
   $p = "C:\Path\To\Folder"
   icacls $p /setowner "NT AUTHORITY\SYSTEM" /T
   ```
   付け過ぎた権限を既定に戻す（継承/ACLの修復）:
   ```cmd
   icacls "C:\Path\To\Folder" /inheritance:e
   icacls "C:\Path\To\Folder" /reset
   ```
3. Defender: 設定と許可アプリ
   - 状態確認:
     ```powershell
     Get-MpPreference | Select EnableControlledFolderAccess, ControlledFolderAccessAllowedApplications
     ```
   - 許可追加:
     ```powershell
     Add-MpPreference -ControlledFolderAccessAllowedApplications "C:\Path\app.exe"
     ```
4. ファイルがロック中なら（見つけ方）
   - Win + R → resmon → CPU タブ → 関連付けられたハンドルでパス検索 → 該当プロセスを終了、または再起動／セーフモードで作業。

⚠ 注意: Defender/EDR の一時無効化や広範な ACL 変更は作業中のみ行うこと。完了後は必ず元に戻す（ロールバック: ポリシーと ACL を復元）。

## よくある落とし穴（やってはいけない）
- Windows/Program Files/System32 直下への広範な所有権変更
- 共有フォルダーに Everyone:フル 付与
- EDR を恒常的に無効化したまま運用すること

## 再発防止チェックリスト
- 標準ユーザー運用＋必要時のみ昇格を徹底する
- インストーラーはダウンロード直後に実行してロックやブロックを回避する
- 変更手順を文書化しレビューを行う
