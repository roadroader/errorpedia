## 要約
- 発生条件（1行）  
  .NET Framework 3.5 の有効化中に必要な SxS ファイルが見つからないか、OS のアクセス/ポリシーでブロックされると 0x800F081F が発生します。  
- 影響（1行）  
  アプリケーションが .NET 3.5 に依存する場合にインストールや実行が不能になります。  
- 緊急度（中、理由1行）  
  中 — 影響はアプリ依存度により大きくなるが、コマンドで短時間に原因切り分け可能なため。

## よくある原因（優先度順）
1. インストールソース（Windows Update/ローカルの sources\sxs）が利用不可または不一致。  
2. WSUS/GPO による機能ダウンロードの制限（/LimitAccess が必要）。  
3. コンポーネントストアの破損（DISM/SFC で検出）。  
4. ファイル/フォルダー権限やロック（インストーラーが SxS を読み取れない）。  
5. AppLocker / ソフトウェア制限ポリシー(SRP) / SmartScreen などの実行制御によるブロック。

## 60秒で確認（最短手順）
- 管理者で PowerShell/コマンドプロンプトを起動 → 再試行  
- 現在のACLを確認：
```cmd
icacls "C:\Windows\winsxs"
```
- 自分の特権を確認：
```cmd
whoami /groups & whoami /priv
```
- Defenderのブロック/履歴とCFA有効状態を確認：
```powershell
Get-MpThreatDetection -ErrorAction SilentlyContinue | Select Timestamp, Action, Resources | Select -First 5
Get-MpPreference | Select EnableControlledFolderAccess
```
- TEMPフォルダーの権限を確認：
```cmd
echo %TEMP%
icacls "%TEMP%"
```
- GPOの影響をスナップショット：
```cmd
gpresult /h "%USERPROFILE%\Desktop\gp.html"
```

## 5分で解決（実務フロー）
1) インストールソースの確認と指定  
   - ISO/インストールメディアから sources\sxs を用意し、以下で実行：
```cmd
dism /online /enable-feature /featurename:NetFx3 /All /Source:D:\sources\sxs /LimitAccess
```
2) コンポーネントストア修復（問題が疑われる場合）：
```cmd
dism /online /cleanup-image /restorehealth
sfc /scannow
```
3) 所有権/権限が原因なら付与：
```cmd
takeown /F "C:\Windows\winsxs" /R /D Y
icacls "C:\Windows\winsxs" /grant Administrators:F /T
```
   ⚠ 注意: 所有権/ACL を広範に変更するとシステムが不安定になります。作業後に元に戻す（ロールバック例を参照）。  
4) 制御されたフォルダーアクセスやEDRがブロックしている場合は一時許可を追加または一時無効化し、インストーラーを実行。  
   - ブロック解除（ファイルの Zone 情報解除）:
```powershell
Unblock-File -Path "C:\Path\to\installer-or-script.exe"
```
   ⚠ 注意: 一時無効化は作業中のみ行う。作業完了後は必ず再有効化（ロールバック：元の設定に戻す）。

## 根本対策（恒久策）
- インストールを行う標準手順に sources\sxs の準備を明文化する。  
- Defender/EDR に対してインストーラーの例外登録ルールを作成する。  
- GPO/WSUS のポリシー確認と、必要時に機能のダウンロードを許可する例外フローを整備する。  
- 共有/ZIP 直実行を避け、必ずローカル（例: C:\Temp）へコピーしてから実行する。

## 詳細手順（Windows 10/11）
1. GUI での所有者と継承の確認（対象: C:\Windows\winsxs または使用するソースフォルダー）。  
2. コマンド例（管理者で実行）:
```cmd
takeown /F "C:\Windows\winsxs" /R /D Y
icacls "C:\Windows\winsxs" /grant Administrators:F /T
```
   ロールバック例（SYSTEM に戻す）:
```powershell
$p = "C:\Windows\winsxs"
icacls $p /setowner "NT AUTHORITY\SYSTEM" /T
```
   付け過ぎた権限を既定に戻す（継承/ACL の修復）:
```cmd
icacls "C:\Windows\winsxs" /inheritance:e
icacls "C:\Windows\winsxs" /reset
```
3. Defender: 状態確認と許可追加  
```powershell
Get-MpPreference | Select EnableControlledFolderAccess, ControlledFolderAccessAllowedApplications
Add-MpPreference -ControlledFolderAccessAllowedApplications "C:\Path\to\dism.exe"
```
   ロールバック: Add-MpPreference で追加した許可は削除またはポリシーで管理を元に戻す。  
4. ファイルがロック中なら（見つけ方）  
   - Win + R → resmon → CPU タブ → 関連付けられたハンドルでパス検索 → 該当プロセスを終了、または再起動／セーフモードで作業。  
   ⚠ 注意: プロセス強制終了はサービス停止やデータ損失を招く可能性あり。ロールバック: 再起動でサービスを復帰させる。

## よくある落とし穴（やってはいけない）
- Windows フォルダー直下（winsxs 等）に対して大規模に Everyone:Full を付与する。  
- 影響範囲を確認せずに Defender/EDR を恒久的に無効化する。

## 再発防止チェックリスト
- 標準ユーザー運用を基本とし、必要時のみ昇格を許可する。  
- インストーラーはダウンロード直後に実行し、ZIP/ネットワーク上での直接実行を避ける。  
- 変更内容を記録し、権限操作は最小権限で実施する。
